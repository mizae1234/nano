# 🤖 น้องนาโน (Nano Ticket) — ระบบ Service Ticket อัจฉริยะ (LINE & AI Integration)

"น้องนาโน" คือ ระบบ Helpdesk / IT Service Ticket อัจฉริยะในรูปแบบ Multi-Tenant ที่ออกแบบมาเพื่อปฏิวัติการแจ้งปัญหาและการติดตามงานในองค์กร โดยเชื่อมต่อการทำงานระหว่างหน้าเว็บแอปพลิเคชัน (Next.js Dashboard) เข้ากับแอปพลิเคชันแชทยอดนิยมอย่าง **LINE (ผ่าน LINE OA & LIFF)** และเสริมประสิทธิภาพการวิเคราะห์ด้วย **Google Gemini AI (LLM)**

---

## 🌟 1. จุดเด่นหลักของระบบ (Key Capabilities)

### 🤖 1.1 การสร้างตั๋วงานอัจฉริยะผ่านภาษาธรรมชาติ (AI NLP Ticket Creation)
* **พิมพ์แจ้งปัญหาอิสระ:** ผู้ใช้งานไม่จำเป็นต้องกรอกฟอร์มซับซ้อน เพียงแค่พิมพ์เล่าปัญหาเป็นภาษาเขียนทั่วไปในไลน์ เช่น *"แจ้ง [ecommerce] หน้าชำระเงินค้าง เงินตัดแล้วแต่ระบบไม่บันทึก"*
* **การคัดแยกโดย AI:** ระบบจะส่งข้อความไปให้ **Gemini AI** วิเคราะห์เพื่อ:
  1. แปลงเป็นข้อความหัวข้อ (Title) ที่กระชับและเข้าใจง่าย
  2. จัดประเภทประเภทตั๋ว (เช่น `BUG`, `FEATURE`, `QUESTION`, `TASK`)
  3. เลือกหมวดหมู่ (Category) และแผนกที่ดูแลให้อัตโนมัติ
  4. ประเมินระดับความสำคัญ (Priority) เช่น หากพบคำวิกฤตจะปรับเป็น `URGENT` ทันที

### 🔮 1.2 การคิวรีวิเคราะห์และทำนายแนวโน้มด้วย AI (AI-Powered Analytics Queries)
* **คิวรีฐานข้อมูลสดผ่าน AI:** ผู้ใช้และผู้บริหารสามารถพิมพ์ถามคำถามเพื่อวิเคราะห์ข้อมูลแบบสถิติเรียลไทม์ได้โดยตรงผ่านแชทไลน์ เช่น
  * *"วิเคราะห์แนวโน้มปัญหาของเดือนนี้ให้หน่อย"*
  * *"ช่วงนี้มีตั๋วค้างในระบบประเภทไหนมากที่สุด พร้อมคำแนะนำ"*
  * *"สรุปงานแผนก IT ที่ยังไม่คืบหน้า"*
* **วิเคราะห์เชิงรวบรวม (Semantic Snapshot):** AI จะดึงโครงสร้างข้อมูลปัจจุบัน (Database Snapshot) มาประมวลผล สรุปเป็นรายงานรายข้อที่อ่านเข้าใจง่าย พร้อมให้ข้อเสนอแนะเชิงคาดการณ์เพื่อป้องกันคอขวดของงาน

### 💬 1.3 การทำงานและแจ้งเตือนผ่านกลุ่มไลน์ (Group Chat Collaboration)
* **บอทช่วยประสานงานในกลุ่ม:** สามารถเชิญ "น้องนาโน" เข้ากลุ่มไลน์ประสานงานทั่วไปได้ 
* **คำสั่งและกลุ่มงานเฉพาะ:** เมื่อมีตั๋วงานใหม่เกิดขึ้นหรือได้รับการแก้ไขเสร็จสิ้น ระบบจะทำการ **Push Notification** ในรูปแบบ **Flex Message** ที่สวยงามเข้าไปยังกลุ่มไลน์ของแผนกที่เกี่ยวข้องโดยตรง เพื่อให้เจ้าหน้าที่กดรับงานหรือมอนิเตอร์ได้ทันที
* **คำสั่งย่อในกลุ่ม:** พิมพ์สั่งงานบอทในกลุ่มได้ง่ายๆ (โดยนำหน้าด้วยคีย์เวิร์ดบอท เช่น *"นาโน สรุปเดือนนี้"*, *"นาโน ดูตั๋ว"*, *"นาโน แจ้ง..."*)

### 👤 1.4 ระบบโปรไฟล์พนักงานเดี่ยวผ่าน LIFF (LIFF Profile & Standalone WebView)
* **ความสะดวกบนมือถือ:** พิมพ์คำว่า `"โปรไฟล์"` หรือ `"ลงทะเบียน"` ในแชทส่วนตัว (DM) บอทจะตอบกลับด้วยการ์ดโปรไฟล์ส่วนตัว (Profile Flex Card)
* **Standalone WebView:** เมื่อกดปุ่ม "แก้ไขโปรไฟล์" ระบบจะเปิดหน้าจอบน **LINE LIFF (LINE Front-end Framework)** เพื่อให้พนักงานอัปเดต รหัสพนักงาน ชื่อ-นามสกุล และเลือกแผนกของตนเอง โดยไม่ต้องสลับหน้าจอไปที่แอปพลิเคชันอื่น
* **ปลอดภัยสูง:** หน้าจอดังกล่าวมีระบบป้องกันข้อมูลความลับ โดยการแสดงผลบัตรโปรไฟล์นี้จะจำกัดให้ใช้งานและเห็นได้ **เฉพาะในห้องแชทส่วนตัวกับบอท (DM) เท่านั้น** จะไม่แสดงผลในการ์ดกลุ่มแชท

## 🔒 2. ระบบสิทธิ์และการนำทางอัจฉริยะ (Role-Based Routing)

ระบบมีฟีเจอร์การดึงตั๋ว 3 รายการล่าสุดมาแสดงเป็น Carousel Flex Message พร้อม **ปุ่มดูตั๋วทั้งหมดบนเว็บ** ที่มีความฉลาดในการวิเคราะห์บทบาท (Role) ของพนักงานและเปลี่ยนปลายทางลิงก์นำทางให้อัตโนมัติ (Role-Based Deep Linking) หลังการทำ Single-Sign-On (SSO):

1. **ผู้ใช้งานทั่วไป (USER):**
   * **ในไลน์บอท:** มองเห็นและค้นหาได้ **เฉพาะตั๋วที่ตนเองสร้างขึ้น** เท่านั้น
   * **ปุ่มนำทางไปหน้าเว็บ:** นำทางไปยังหน้า `/ticket` (ตั๋วงานของฉัน) เพื่อให้ดูสถานะประวัติการส่ง
2. **เจ้าหน้าที่ปฏิบัติงาน (IT / Agent):**
   * **ในไลน์บอท:** เห็นตั๋วที่ตนรับผิดชอบ + ตั๋วที่สร้าง + **ตั๋วใหม่ของแผนกที่ยังไม่มีเจ้าหน้าที่รับงาน (Unassigned)** เพื่อกดรับเคลมงานได้สะดวก
   * **ปุ่มนำทางไปหน้าเว็บ:** นำทางไปยังหน้า `/it` (IT Board / Kanban Board สำหรับขยับสถานะและจัดการงานของแผนก)
3. **หัวหน้าแผนก (DEPT_ADMIN):**
   * **ในไลน์บอท:** เห็นข้อมูลตั๋วงานและสถิติทั้งหมดที่เกิดขึ้นภายในแผนกสังกัดของตน
   * **ปุ่มนำทางไปหน้าเว็บ:** นำทางไปยังหน้า `/dept` (แผนกของฉัน เพื่อดูสรุปข้อมูลงาน คัดแยก และแจกจ่ายงานให้ลูกน้องในแผนก)
4. **ผู้ดูแลระบบ (ADMIN / SUPER_ADMIN):**
   * **ในไลน์บอท:** เห็นตั๋วและสามารถสั่งยิงข่าวสารประกาศ (Broadcast Flex Message) หาพนักงานหรือกลุ่มไลน์ทั้งหมดได้
   * **ปุ่มนำทางไปหน้าเว็บ:** นำทางไปยังหลังบ้านสำหรับตั้งค่าการเชื่อมต่อระบบและวิเคราะห์รายงานสรุปภาพรวมทั้งหมด

## 📈 3. ระบบรายงานและการจัดการข้อมูลประสิทธิภาพ (Performance Reports)
* **ส่งออกรายงานผ่านหน้าเว็บ:** ระบบรองรับการแปลงและดึงข้อมูลรายงานตั๋วงานและประสิทธิภาพในการแก้ไขปัญหา (SLA Hours)
* **Native Excel (.xlsx) Export:** ส่งออกข้อมูลตั๋วงานเป็นไฟล์ **`.xlsx` แท้โดยตรง** (ไม่ใช่ไฟล์ CSV แบบเก่า) ผ่านไลบรารีประสิทธิภาพสูง หมดปัญหาตัวอักษรภาษาไทยเพี้ยนหรือรูปแบบตกหล่นเมื่อเปิดใช้งานใน Microsoft Excel

## 🛠️ 4. สถาปัตยกรรมและเทคโนโลยีที่ใช้ (Tech Stack)

* **Core Web Framework:** Next.js 14 (App Router) พร้อมโครงสร้างสไตล์ Vanilla CSS เพื่อความยืดหยุ่นและดีไซน์หรูหราแบบพรีเมียม
* **Database & ORM:** PostgreSQL + Prisma (v7) 
* **Multi-Database Support:** ออกแบบโครงสร้างรองรับทั้ง **Shared Database** (ระดับปกติ) และ **Dedicated Database** (แยก Database เฉพาะ สำหรับลูกค้าระดับ Enterprise เพื่อความปลอดภัยสูงสุด)
* **LINE Integration:** LINE Messaging API (Webhook & Push/Reply Message) และ LINE LIFF SDK (v2)
* **Encryption System:** เข้ารหัสความลับผ่าน **AES-256-GCM** สำหรับจัดเก็บ Channel Access Token และ Channel Secret ของระบบ/Tenant อย่างปลอดภัยสูงสุดในฐานข้อมูล
* **AI Engine:** Google Generative AI SDK (ขุมพลัง Google Gemini Flash)
* **Deployment:** Docker & Docker Compose (Containerization) รันบน Linux Server
